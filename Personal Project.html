PROJECT AKHIR PENAMBANGAN DATA#

Klasifikasi dataset “Vertebral Column”#

Dataset “Vertebral Column” merupakan kumpulan data yang digunakan untuk klasifikasi kesehatan tulang belakang. Tujuan dari klasifikasi ini adalah untuk mengidentifikasi apakah kondisi tulang belakang seseorang normal atau mengalami masalah tertentu.

MEMAHAMI DATA#

Pada tahap ini meliputi empat cara untuk memahami data, yaitu memahami cara memperoleh data, eksplorasi data, mengetahui tipe data pada setiap kolom, dan mengidentifikasi kualitas data.

1. Memahami cara memperoleh data#

Dataset Vertebral Column ini bagian dari UCI Machine Learning Repository, yang merupakan kumpulan basis data yang banyak digunakan untuk studi empiris algoritma pembelajaran mesin. Dataset Vertebral Column berisi informasi tentang fitur biomekanik dari kolom vertebral, dan digunakan untuk mengklasifikasikan pasien ortopedi ke dalam berbagai kategori berdasarkan fitur-fitur ini. Dataset ini terdiri dari fitur-fitur biomekanik yang berasal dari bentuk dan orientasi pelvis dan tulang belakang lumbar.

Dataset Vertebral Column ini dapat diunduh dari UCI Machine Learning Repository menggunakan tautan berikut https://archive.ics.uci.edu/dataset/212/vertebral+column

Pada dataset ini terdapat 310 data dan berisi enam fitur, yaitu:

  • pelvic_incidence: Sudut antara garis tegak lurus ke lempeng sakral dan garis yang menghubungkan lempeng sakral ke pusat sumbu bicoxofemoral.

  • pelvic_tilt: Sudut antara garis vertikal dan garis yang menghubungkan lempeng sakral ke pusat sumbu bicoxofemoral.

  • lumbar_lordosis_angle: Sudut antara tulang belakang lumbar atas dan bawah.

  • sacral_slope: Sudut antara garis horizontal dan lempeng sakral.

  • pelvic_radius: Jarak dari pusat sumbu bicoxofemoral ke pusat lempeng sakral.

  • degree_spondylolisthesis: Mengukur perpindahan satu vertebra relatif terhadap vertebra lainnya.

2. Eksplorasi data#

Setelah memuat dataset, langkah selanjutnya adalah melakukan eksplorasi data. Eksplorasi data adalah tahap awal dalam analisis data yang bertujuan untuk memahami karakteristik, pola, dan sifat dasar dari dataset. Tujuan utama dari eksplorasi data adalah untuk mendapatkan wawasan yang berguna dari data yang tersedia sebelum melakukan analisis yang lebih mendalam atau membangun model.

Eksplorasi data ini melibatkan beberapa langkah penting:

  1. Jumlah (Count): Ini menunjukkan berapa banyak entri data yang ada untuk setiap variabel. Dalam kasus ini, Anda memiliki 900 entri untuk setiap variabel.

  1. Rata-rata (Mean): Ini adalah nilai rata-rata dari seluruh data untuk setiap variabel. Rata-rata memberi Anda gambaran tentang nilai tengah dari distribusi data.

  1. Standar Deviasi (Standard Deviation): Standar deviasi mengukur seberapa jauh titik data tersebar dari rata-rata. Semakin tinggi nilai standar deviasi, semakin besar variasi dalam data.

  1. Minimum: Ini adalah nilai terkecil yang ada dalam dataset untuk setiap variabel. Ini memberi Anda gambaran tentang rentang nilai yang mungkin untuk setiap variabel.

  1. Persentil ke-25 (Q1): Ini adalah nilai yang membagi 25% data terendah dari yang lainnya. Dalam konteks ini, 25% dari data memiliki nilai di bawah nilai ini.

  1. Persentil ke-50 (Median atau Q2): Median adalah nilai tengah dari dataset saat diurutkan dari yang terkecil hingga yang terbesar. Dalam hal ini, 50% dari data memiliki nilai di bawah median.

  1. Persentil ke-75 (Q3): Ini adalah nilai yang membagi 75% data terendah dari yang lainnya. Dalam konteks ini, 75% dari data memiliki nilai di bawah nilai ini.

  1. Maksimum: Ini adalah nilai terbesar yang ada dalam dataset untuk setiap variabel. Ini memberi Anda gambaran tentang rentang nilai yang mungkin untuk setiap variabel.

B. Pemeriksaan LOCAL OUTLIER FACTOR (LOF)#

Langkah berikutnya adalah deteksi outlier. Deteksi outlier adalah metode untuk mengidentifikasi titik-titik data yang dianggap sebagai outlier atau anomali. Outlier adalah nilai yang secara signifikan berbeda dari sebagian besar data lainnya dalam sebuah dataset. Mengidentifikasi outlier bermanfaat untuk meningkatkan akurasi analisis dan model prediktif.

PERHITUNGAN LOCAL OUTLIER FACTOR (LOF) MENGGUNAKAN PYTHON#

Untuk perhitungan LOF menggunakan Python ini dapat menggunakan library scikit-learn. Cara ini mempermudah dan mempercepat proses deteksi outlier. Berikut adalah kode untuk menghitung LOF:

import pandas as pd
from sklearn.neighbors import LocalOutlierFactor
import matplotlib.pyplot as plt
import numpy as np

# Baca dataset
df = pd.read_csv('/work/Vertebral_Dataset.csv')

# Siapkan data untuk LOF (drop kolom 'Class' karena fitur kategorikal)
x = df.drop('Class_att', axis=1)

# Inisialisasi model LOF
lof = LocalOutlierFactor(n_neighbors=20, contamination=0.040)

# Terapkan LOF pada dataset
y_pred = lof.fit_predict(x) # fit_predict(x): Menerapkan model LOF pada dataset x dan mengembalikan prediksi. Hasil prediksi berupa label -1 untuk outlier dan 1 untuk inlier.
# Skor anomali (semakin rendah, semakin besar kemungkinan outlier)
anomaly_scores = lof.negative_outlier_factor_ # negative_outlier_factor_: Menghasilkan skor anomali untuk setiap sampel dalam dataset. Semakin rendah nilainya, semakin besar kemungkinan sampel tersebut adalah outlier.
# Tambahkan kolom hasil prediksi dan skor anomali ke dalam dataframe
df['LOF_Prediksi'] = y_pred # Menambahkan kolom LOF_Prediksi ke DataFrame df yang berisi hasil prediksi LOF (1 untuk inlier, -1 untuk outlier).
df['LOF_Skor_Anomali'] = anomaly_scores # Menambahkan kolom LOF_Skor_Anomali yang berisi skor anomali dari LOF.

# Visualisasi hasil (opsional)
plt.figure(figsize=(10, 6)) # Membuat figure baru dengan ukuran 10x6.
plt.title("LOF Anomaly Scores") # Memberi judul pada plot.
plt.scatter(np.arange(len(anomaly_scores)), anomaly_scores, c=(y_pred == -1), cmap='coolwarm', edgecolor='k') # Membuat scatter plot dari skor anomali.
plt.axhline(y=-1.5, color='r', linestyle='--')  # Menambahkan garis horizontal pada y = -1.3 untuk memisahkan outlier (opsional, dapat disesuaikan dengan konteks data).
plt.xlabel("Indeks")
plt.ylabel("Skor Anomali LOF")
plt.show()

ENSEMBLE LEARNING#

Setelah dilakukan Pemodelan, ada banyak cara untuk meningkatkan akurasi prediksi dari model yang telah kita buat salah satunya adalah Ensemble Learning.

Ensemble learning adalah teknik yang digunakan untuk meningkatkan kinerja dan akurasi model-model pembelajaran mesin dengan menggabungkan hasil prediksi dari beberapa model atau algoritma yang berbeda.

Istilah ensemble merujuk pada konsep menggabungkan banyak elemen menjadi satu kesatuan. Konsep penggabungan berbagai pendekatan pembelajaran mesin diharapkan dapat mengatasi kelemahan individu dari masing-masing model dan menciptakan prediksi yang lebih akurat dan stabil.

Cara kerja dari metode ensemble melibatkan penggunaan beberapa model yang beragam dalam satu tim. Hasil prediksi dari masing-masing model tersebut digabungkan untuk membuat keputusan akhir.

Pada project kali ini saya akan menggunakan 2 metode dalam ensemble learning yaitu Stacking dan Bagging dan mencari metode mana yang terbaik untuk data kita.

METODE STACKING#

1. Pengertian Metode Stacking#

Stacking adalah sebuah metode dimana sebuah model ditraining untuk menggabungkan beberapa individual model yang disebut first-level learner, sedangkan yang digabungkan disebut second-level learner atau meta-learner.

Stacking melibatkan dua atau lebih base learner atau model dasar sebagai model level-1, dan meta-learner yang menggabungkan prediksi base learner disebut sebagai level-2. Model yang berbeda digunakan oleh base learner untuk belajar dari suatu dataset. Output dari masing-masing model base learner dikumpulkan untuk membuat dataset baru. Di dalam dataset yang baru, setiap sampel berhubungan dengan nilai pada dataset awal yang seharusnya diprediksi. Selanjutnya dataset tersebut digunakan oleh model meta-learner hingga mendapatkan hasil akhir.

METODE BAGGING#

Bagging (bootstrap aggregating) merupakan metode ensemble yang melibatkan pelatihan beberapa model secara sendiri pada dataset yang diacak, dan menggabungkan prediksi mereka melalui pemungutan suara.

setiap model dilatih pada dataset yang diacak dari data yang diambil sampelnya,  data individual dapat dipilih lebih dari satu kali. data acak ini dikenal sebagai sampel bootstrap. Dengan melatih model pada bootstrap yang berbeda, bagging mengurangi varians dari masing-masing model. Bagging juga dapat menghindari overfitting.

Prediksi dari semua model yang diambil sampelnya kemudian digabungkan dan dibuat voting yang dimana hasil prediksi terbanyak akan di keluarkan sebagai hasil prediksi akhir

Bagging sangat efektif dalam mengurangi varians dan overfitting, membuat model lebih tangguh dan akurat, terutama dalam kasus di mana masing-masing model rentan terhadap variabilitas tinggi.

  • Pertama,fungsi akan dibuat dengan nama bagging dan mengambil parameter x_train, y_train,x_test,y_test,jumlah_juri

  • Kedua, melakukan perulangan yang dimulai dari i = 1 sampai jumlah_juri

  • Keempat, lakukan fit (latih) dengan x_trian dan y_train dari inputan parameter kita sebelumnya

  • Kelima, lakukan prediksi x_test dengan model yang kita latih sebelumnya sesuai dengan n_estimator

  • Keenam, Lalu hasil dari prediksi kita Encoder ke tipe numeric agar bisa kita hitung akurasinya

  • Ketujuh, memasukkan data ke array accuracy yang kita akan gunakan sebagai pengelompokan accuracy dari beberapa n_estimator

  • Kedelapan, memilih tingkat akurasi tertinggi dan jumlah n_estimator terendah

from sklearn.svm import SVC
from sklearn.naive_bayes import GaussianNB
from sklearn.ensemble import BaggingClassifier
from sklearn.datasets import make_classification
from sklearn.metrics import accuracy_score
import pickle


def bagging(x_train, y_train,x_test,y_test,jumlah_juri):
    accuracy = []
    # menentukan rentang mulai dari 1 hingga jumlah_juri (termasuk) dengan langkah penambahan sebesar 1 pada setiap iterasi.
    for i in range(1,jumlah_juri + 1, 1): 
        clf = BaggingClassifier(base_estimator=GaussianNB(),n_estimators=i, random_state=0).fit(x_train, y_train)
        y_pred = clf.predict(x_test)
        le = LabelEncoder()
        y_pred_enc = le.fit_transform(y_pred)
        acc = accuracy_score(y_test, y_pred_enc)
        accuracy.append(acc)
    return accuracy
